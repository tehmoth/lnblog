the IDN non-issue

apparently there's a new phishing attack out there that uses modern browsers' (except IE heh) support for the IDN (international characters in URL support) standard. This involved replacing latin characters with unicode characters with similar appearance. This is only a problem if you can't tell the difference between these two characters:

a а

so people disabling IDN support in your browsers need to instead buy glasses, I would rather not limit the functionality of my browser when i still have fully functioning eyes.

(yes i know, there was a bug that the mozilla preference disableIDN didn't work when you set it, but I wouldn't have used it anyway).
